# PROJECT_PLAN.md

## 项目名称
水下目标检测网页系统开发计划

---

## 一、项目目标
构建一个网页端的水下目标检测系统，实现：
- 图片检测
- 视频检测
- 检测结果展示
- 视频分析统计
- 结果下载

---

## 二、当前基础
- 已有训练好的目标检测模型
- 模型为 PyTorch `.pt` 权重文件
- 项目托管在 GitHub
- 后续主要通过 Cursor 的 Codex 插件开发

---

## 三、第一阶段目标（MVP）

### 1. 图片检测
- 上传图片
- 执行目标检测
- 返回带框图片
- 返回类别、数量、置信度

### 2. 视频检测
- 上传视频
- 逐帧检测
- 生成带框视频
- 返回视频处理结果

### 3. 视频分析功能
- 每类目标出现的总帧数
- 按时间轴显示目标出现区间
- 每秒目标数量变化曲线

### 4. 前端展示
- 图片检测页面
- 视频检测页面
- 统计分析展示区域
- 结果下载按钮

---

## 四、阶段划分

### 阶段 1：项目骨架搭建
- 建立项目目录
- 建立后端基础结构
- 建立前端基础结构
- 配置环境变量
- 完成基础 README 与开发规范

### 阶段 2：后端核心能力
- 模型加载模块
- 图片检测服务
- 视频检测服务
- 结果图绘制
- 结果视频输出
- JSON 元数据保存

### 阶段 3：视频分析能力
- 每类目标帧数统计
- 目标出现时间区间统计
- 每秒目标数量曲线统计

### 阶段 4：前端页面实现
- 首页
- 图片检测页
- 视频检测页
- 图表分析展示

### 阶段 5：联调与优化
- 前后端接口联调
- 错误处理
- 参数调整
- 页面优化
- 下载功能完善

## 七、交付标准
第一版完成后，系统至少应满足：

1. 网页端可上传图片并完成检测
2. 网页端可上传视频并生成检测后视频
3. 页面可展示统计分析结果
4. 用户可下载检测结果
